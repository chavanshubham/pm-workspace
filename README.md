# PM Knowledge Workspace

A personal knowledge management system for product managers. Notes are Markdown with YAML frontmatter. Claude Code skills automate note creation, summarization, and index generation.

## Structure

```
.
├── daily/              # One file per day (YYYY-MM-DD.md) — local only
├── meetings/           # One file per meeting (YYYY-MM-DD-title-slug.md) — local only
├── projects/           # One living document per project — local only
├── people/             # One running notes file per person — local only
├── _index/             # Auto-generated by /sync — local only, never edit manually
├── _templates/         # Templates used by skills
└── .claude/skills/     # Claude Code skills
```

## Skills

| Command | What it does |
|---------|-------------|
| `/daily` | Create or open today's daily note |
| `/meeting <title> [attendees] [project]` | Create a structured meeting note |
| `/project-note <name>` | Create or open a project living document |
| `/person <name>` | Create or open a person profile note |
| `/mom <meeting-title-or-date>` | Generate shareable Minutes of Meeting |
| `/email <person> [subject]` | Draft a professional email grounded in your notes |
| `/status <project>` | Generate a project status report |
| `/standup` | Generate a standup from yesterday's notes |
| `/week [YYYY-MM-DD]` | Generate a weekly recap |
| `/sync` | Rebuild all index files from frontmatter |

## Action Item Lifecycle

Action items are created in meeting frontmatter:

```yaml
action_items:
  - owner: person-slug
    task: "Do the thing"
    due: 2026-04-10
    status: open
```

When a follow-up meeting closes an item, add a `resolves:` block to that meeting note — no editing of the original required:

```yaml
resolves:
  - source_meeting: 2026-04-06-meeting-slug
    owner: person-slug
    task: "Do the thing"
    outcome: done         # done | partial | carried-over
    note: "Resolved in review session"
```

Run `/sync` and the item moves from `_index/open-actions.md` to the resolved section in the project and person indexes. The full lifecycle is visible in `_index/all-actions.md`.

## Indexes (auto-generated by `/sync`)

| File | Contents |
|------|----------|
| `_index/open-actions.md` | All open/partial/carried-over items across all projects, sorted by due date |
| `_index/all-actions.md` | Full lifecycle view grouped by project — open and resolved |
| `_index/by-project/<slug>.md` | Isolated project view: meetings, daily notes, open + resolved actions |
| `_index/by-person/<slug>.md` | Per-person view: projects, meetings, action items grouped by project |
| `_index/daily-combined/<date>-combined.md` | Merged daily journal + meetings for a given day |

## Workflow

1. Create or update any note using a skill.
2. Keep `projects:` and `people:` frontmatter accurate.
3. Run `/sync` — all indexes are fully regenerated.

`/sync` is idempotent and always safe to re-run.
