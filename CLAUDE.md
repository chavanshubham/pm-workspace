# PM Knowledge Workspace

This is a product manager's knowledge management system. All notes are Markdown with YAML frontmatter. Claude Code skills automate note creation, summarization, and aggregation.

## Workspace Layout

| Directory | Purpose |
|-----------|---------|
| `projects/` | One living document per project (update continuously) |
| `people/` | One running notes file per person |
| `daily/` | One file per calendar day (`YYYY-MM-DD.md`) |
| `meetings/` | One file per meeting (`YYYY-MM-DD-title-slug.md`) |
| `_index/by-project/` | AUTO-GENERATED — all notes tagged with a project |
| `_index/by-person/` | AUTO-GENERATED — all notes tagged with a person |
| `_index/daily-combined/` | AUTO-GENERATED — merged view for a given day |
| `_index/open-actions.md` | AUTO-GENERATED — all open action items sorted by due date |
| `_templates/` | Templates used by skills — do not edit unless updating the system |

**Never edit `_index/` files manually.** They are fully regenerated by `/sync`.

---

## Frontmatter Schema

Every note must have these fields:

```yaml
type: daily | meeting | project | person
title: "Human-readable title"
date: 2026-04-06          # ISO 8601 date of the note/event
created: 2026-04-06T09:00
updated: 2026-04-06T14:30
projects:                 # kebab-case slugs — each maps to projects/<slug>.md
  - alpha-redesign
people:                   # kebab-case slugs — each maps to people/<slug>.md
  - sarah-kim
tags:                     # free-form tags
  - ux
```

### Meeting notes also require:
```yaml
meeting_duration: 60      # minutes
location: ""
action_items:             # MUST be structured YAML — not prose — for /sync to extract them
  - owner: sarah-kim
    task: "Share wireframes"
    due: 2026-04-10
    status: open          # open | done
decisions:
  - "Use design system v2 as the baseline"
```

### Project notes also require:
```yaml
slug: alpha-redesign      # kebab-case — must match filename
status: active            # active | paused | done | archived
owner: sarah-kim
target_date: 2026-06-30
```

### Person notes also require:
```yaml
slug: sarah-kim           # kebab-case — must match filename
role: "Head of Design"
team: "Product Design"
```

---

## Slug Convention

Slugs are always kebab-case and map directly to filenames:
- `alpha-redesign` → `projects/alpha-redesign.md` + `_index/by-project/alpha-redesign.md`
- `sarah-kim` → `people/sarah-kim.md` + `_index/by-person/sarah-kim.md`

Use only slugs that have (or will have) a corresponding file. `/sync` will warn about orphaned slugs.

## Available Skills

| Command | What it does |
|---------|-------------|
| `/daily` | Create or open today's daily note |
| `/meeting <title> [attendees] [project]` | Create a structured meeting note |
| `/project-note <name>` | Create or open a project living document |
| `/person <name>` | Create or open a person profile note |
| `/mom <meeting-title-or-date>` | Generate shareable Minutes of Meeting from a meeting note |
| `/email <person> [subject]` | Draft a professional email to a person (copy-paste ready) |
| `/status <project>` | Generate a project status report (narrative + action items table) |
| `/standup` | Generate a standup from yesterday's notes |
| `/week [YYYY-MM-DD]` | Generate a weekly recap (defaults to current week) |
| `/sync` | Rebuild ALL index files from frontmatter |

---

## "Update One, Update All" Workflow

1. Create or edit any note (daily, meeting, project, person).
2. Ensure `projects:` and `people:` frontmatter are accurate.
3. Run `/sync` — this scans every note's frontmatter and rewrites all `_index/` files.
4. All indexes now reflect the latest state across all notes.

`/sync` is a full regeneration — idempotent, always safe to re-run.

---

## Action Items Must Be Structured YAML

For action items to appear in project indexes and `/status` output, they **must** be in the YAML frontmatter of meeting notes, not just written as prose:

```yaml
action_items:
  - owner: james-liu
    task: "Audit current API endpoints"
    due: 2026-04-09
    status: open
```

The `/mom` skill can help convert prose action items into this format after a meeting.

## File Naming Conventions

- Daily notes: `daily/2026-04-06.md`
- Meeting notes: `meetings/2026-04-06-alpha-kickoff.md`
- Projects: `projects/alpha-redesign.md`
- People: `people/sarah-kim.md`

Alphabetical sort = chronological sort for all date-prefixed files.
